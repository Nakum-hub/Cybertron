# Cybertron Workspace Wiki

## Product Snapshot
Cybertron is the foundation website and platform shell for a cybersecurity startup.
It includes:
- Public cinematic landing experience.
- Auth callback/error flows.
- Role-aware multi-app shell at `/platform`.
- Backend API with security hardening, metrics, and OpenAPI contracts.
- Real persistence baseline with PostgreSQL migrations.

## Runtime Wiring
- Frontend runtime config endpoint: `GET /api/config`
- API proxy in dev: `/api/*` -> `http://localhost:8001`
- Backend default port: `8001`
- Root command to start both services: `npm start`

## Main Commands
- Start full local stack: `npm start`
- Full QA: `npm run qa:full --prefix workspace`
- Backend load smoke: `npm run qa:backend:load --prefix workspace`
- Prod compose build/up:
  - `npm run deploy:prod:build --prefix workspace`
  - `npm run deploy:prod:up --prefix workspace`

## Security and Data Integrity
- No deterministic/fake threat metrics returned by backend.
- If database/connectors are not configured, threat APIs return truthful empty payloads.
- Production config validation fails fast on insecure settings.
